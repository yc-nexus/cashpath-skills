# 矛盾分析法

## 触发条件

用户说 "分析矛盾"、"主要矛盾是什么"、"卡住了"、"先做什么"、"选哪个"、"帮我理一下" 时激活。

## 行为规则

### 第一步：列出所有矛盾

穷举当前情境中的所有对立因素。每个矛盾 = 两种力量的对立，格式：

```
矛盾1：A vs B — [一句话描述对立关系]
矛盾2：C vs D — [一句话描述对立关系]
...
```

### 第二步：识别主要矛盾（只选一个）

从矛盾清单中选出**主要矛盾**。判断标准：

1. **决定性** — 解决了它，其他矛盾会自然松动或被消解
2. **紧迫性** — 不解决它，事情无法推进
3. **杠杆性** — 同样的投入，解决它能带来最大回报

标记主要矛盾为 ⭐，并说明理由。

### 第三步：判断矛盾性质

- **对抗性** — 矛盾双方无法共存，必须二选一（如：继续 vs 放弃）
- **非对抗性** — 矛盾双方可以找到平衡或分阶段解决（如：速度 vs 质量）

### 第四步：给出应对策略

如果是**对抗性**矛盾：直接给出选择建议和决策边界。
如果是**非对抗性**矛盾：给出分阶段方案，先解决什么、后解决什么。

### 第五步：输出格式

```
## 矛盾分析

### 矛盾清单
1. [矛盾1]
2. [矛盾2]
...

### ⭐ 主要矛盾
[主要矛盾描述]

**为什么是它：** [理由，包含决定性/紧迫性/杠杆性至少两点]

### 矛盾性质
[对抗性/非对抗性] — [说明]

### 应对策略
[具体动作，可执行、可衡量]

### 监控提示
- [矛盾是否转化的观察点]
```

## 核心原则

- 主要矛盾只能有一个。如果觉得有两个，说明还没挖到底
- 对抗性矛盾必须果断选择，不要和稀泥
- 非对抗性矛盾要分阶段，不同阶段有不同的主要方面
- 矛盾会转化。上次分析的主要矛盾不一定还是今天的主要矛盾
