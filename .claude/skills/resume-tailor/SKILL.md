# 简历定制师

## 触发条件

用户发送招聘 JD 链接/截图/文本，或说"改简历""帮我匹配这个岗位""投这个""优化简历""JD 分析"时激活。

## 前置条件

本技能需要两份输入：

1. **目标岗位的 JD**（职位描述）——用户发送后自动提取
2. **用户的原始简历**——优先使用用户本次对话中提供的版本；如未提供，询问用户粘贴

## 工作流程

### 第一步：JD 结构化拆解

从招聘 JD 中提取以下信息，格式化为表格：

| 维度 | 内容 |
|------|------|
| 岗位名称 | [提取] |
| 公司/行业 | [提取] |
| 硬性门槛 | [学历/年限/语言/证书等不可妥协的要求] |
| 核心技能 | [按出现频次排序，标注 JD 中出现了几次] |
| 加分项 | [非必需但有优势的技能或经验] |
| 隐藏信息 | [从 JD 措辞推断的团队规模/业务阶段/技术栈偏好等] |

### 第二步：简历-JD 匹配度分析

逐条对比用户简历与 JD 要求，输出匹配度矩阵：

```
要求项           简历现状          匹配度   缺失点
──────────────  ────────────────  ──────  ──────────
Python 3年+     有2年Python经验     🟡 70%  年限差1年，但可强调项目深度
K8s 生产经验    简历中未提及        🔴 0%   需补充或从相关经验推断
跨境电商经验    做过Shopify独立站   🟢 90%  直接匹配，需在措辞上对齐JD关键词
...
```

匹配度标记规则：
- 🟢 ≥80%：直接可用，仅需措辞微调
- 🟡 50-79%：有相关经验但需重写以对齐 JD 关键词
- 🔴 <50%：严重缺失，需要提供替代策略

### 第三步：简历改写方案

针对每个 JD 要求，给出具体的改写建议。每条建议包含：

**原始表述**（简历中现在的写法）
↓
**改写后**（对齐 JD 关键词后的版本）
↓
**为什么这么改**（一句理由，说明如何匹配了 JD 的哪条要求）

额外输出：
- **关键词注入清单**：JD 中的高频词和行业术语，应散点嵌入简历各段
- **量化建议**：原始简历中可补充数字的地方（如"负责 XX"→"负责 XX，月活从 10 万提升到 50 万"）

### 第四步：缺失项应对策略

对于匹配度 🔴 的项目，逐个给出替代方案：

1. **可快速补齐的**（如某个工具/框架）→ 建议学习路径和时间估算
2. **可从相关经验推断的**（如"没做过金融行业但做过高并发支付系统"）→ 给出简历中的转写方案
3. **硬伤**（如学历门槛、工作年限）→ 如实告知，给出一句话面试解释话术

### 第五步：输出完整的改写后简历

基于以上分析，输出一份完整的优化后简历，附带：

- **改动说明**：相比原始简历改了哪些、为什么
- **面试预期**：面试官看到这份简历后最可能追问的 3 个问题
- **投递策略**：建议投递时间、是否内推、HR 触点选择

## 输出格式模板

```
## JD 拆解

[第一步的表格]

## 匹配度总览

匹配率：X/Y（百分比）

[第二步的匹配度矩阵]

## 逐条改写

### 1. [JD要求项]
- 现在：...
- 改为：...
- 理由：...

## 关键词注入清单

- [词1]：[嵌入位置建议]
- ...

## 缺失项应对

| 缺失项 | 策略 | 行动建议 |
|--------|------|----------|
| ... | ... | ... |

## 改写后完整简历

[全文]

## 投递策略

- 面试预警：...
- 最佳投递窗口：...
```

## 核心原则

- **不对简历造假**：不改工作年限、不改职位头衔、不编造项目经验。所有改写只能在真实经历基础上重新措辞和包装
- **量化优先**：能用数字表达的结果一定加数字，不能用数字的加范围
- **关键词对齐但不堆砌**：JD 中的术语要自然融入，不要像 SEO 堆词
- **中文 JD 输出中文简历，英文 JD 输出英文简历**：语言跟随岗位描述
- **如果用户简历与 JD 匹配度总览 <40%，主动提示**：客观告知投递成功率可能较低，建议优先补充关键技能
