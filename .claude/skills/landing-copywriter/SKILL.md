# Landing Page 文案助手

## 触发条件

用户提到"写英文文案""landing page 文案""产品介绍""定价页""hero section""features 文案""CTA 按钮""帮我写 SaaS 网站文案"时激活。

## 前置条件

本技能需要以下输入（缺失则逐一询问）：

1. **产品描述**：你的产品是什么、解决什么问题、目标用户是谁
2. **至少 1 个竞品 URL**（推荐 3 个）：用于提取文案模式和差异化定位
3. **页面类型**：首页 / Features 页 / 定价页 / About 页 / CTA 区域

## 工作流程

### 第一步：竞品文案拆解

分析每个竞品的目标页面，逐项提取：

| 维度 | 竞品 A | 竞品 B | 竞品 C |
|------|--------|--------|--------|
| Hero H1（价值主张） | [原文] | [原文] | [原文] |
| Hero 副标题 | [原文] | [原文] | [原文] |
| Features 组织方式 | [列表式/卡片式/对比式] | ... | ... |
| CTA 用词 | [原文 + 语气分析] | ... | ... |
| Social Proof 形式 | [数字/Logo墙/用户引言] | ... | ... |
| 定价锚点措辞 | [原文] | ... | ... |
| 整体语气 | [正式/亲切/技术感/极简/...] | ... | ... |

**关键发现**（完成表格后输出）：
- 三个竞品的共同模式（所有竞品都做的事=行业标准，必须做）
- 三个竞品都没做的事（差异化机会）
- 最有效的写法（哪个竞品的某段文案最打动人？为什么？）

### 第二步：高频词与句式提取

从竞品文案中提取：

**高频词 Top 10**（带频次）：
```
单词/短语          频次    使用场景
──────────────    ────    ──────────
seamless          8       Features 描述，强调"无缝集成"
boost             6       Hero/CTA，强调"提升效率"
...
```

**高频句式模板**（提取可复用的句式骨架）：
```
[Verb] your [noun] without [pain point]
→ "Build your API without writing boilerplate"
→ "Ship your app without worrying about infrastructure"
```

**行业惯用语**：
- 列出该赛道中必须出现的关键词（如 AI 产品必须有 "AI-powered" 或 "intelligent"）
- 列出该赛道用烂了的词（如 "revolutionary"、"game-changing"），建议避开

### 第三步：你的产品文案生成

基于前两步的分析，生成 **A/B 两版**完整文案：

**A 版（跟随策略）**：对齐行业标准写法，让用户一眼就知道你是什么品类
**B 版（差异策略）**：突出你的独特优势，用不同于竞品的语气和角度

每版覆盖（按页面类型选择）：

| 页面区域 | 包含内容 |
|----------|----------|
| Hero Section | H1 + 副标题 + CTA 按钮文案 + 配图建议 |
| Features Block | 3-5 个功能卡片的标题 + 描述 + 图标建议 |
| Social Proof | 信任信号文案（数字/引用/Logo 墙） |
| Pricing | 套餐名称 + 价格锚点描述 + FAQ 文案 |
| Bottom CTA | 最后的转化钩子 |

### 第四步：逐条解释

对每版文案，解释关键选择：

```
A 版 H1：[文案]
├── 为什么用这个词：[理由，关联竞品分析中的哪个发现]
├── 为什么不用 [替代词]：[理由]
└── 预期用户反应：[判断]
```

### 第五步：A/B 测试建议

- **哪个版本更适合冷启动**（SEO 友好、搜索意图匹配）
- **哪个版本更适合投放**（转化导向、行动感更强）
- **建议先测哪个** + 为什么
- **关键监测指标**：点击率 / 停留时间 / 注册转化

## 输出格式模板

```
## 竞品拆解

[第一步表格 + 关键发现]

## 高频词与句式

### 高频词 Top 10
[表格]

### 可复用的句式
- [句式1] → 示例：[改写示例]

### 该避开的词
- [词]：[原因]

## A 版文案（跟随策略）

### Hero
H1: [文案]
副标题: [文案]
CTA: [文案]

### Features
...（按页面类型输出全部区域）

## B 版文案（差异策略）

[同上结构]

## 选择理由

[逐条解释 A/B 版的关键选择]

## A/B 测试建议

- 冷启动推荐：A 版（原因）
- 投放推荐：B 版（原因）
- 先测：...
```

## 核心原则

- **不造生词**：所有用词必须能在竞品或同类产品中找到先例（B 版可适度放宽）
- **一句话能说清就别用两句**：Hero H1 不超过 10 个单词
- **CTA 用动词开头**：Get / Start / Try / Build / Ship，不用名词或形容词
- **数字比形容词有力量**："10x faster" > "blazingly fast"
- **不编造竞品信息**：如果竞品数据不在页面上，不猜测
- **中文产品给中文翻译对照**：输出英文文案的同时，附中文释义
