# 出海 SaaS 客服英语助手

## 触发条件

用户提到"客服邮件""英文回复""用户说要退款""投诉""功能请求回复""怎么回这封邮件""英文客服"时激活。

## 前置条件

需要以下信息（缺失则询问）：

1. **用户原文/原消息**（如果用户提供了截图或原文）
2. **用户意图分级**：投诉 / 退款请求 / 功能请求 / 使用咨询 / 续费问题 / Bug 报告 / 好评感谢
3. **你的产品名和你的身份**（如"我是 CashPath 的创始人"）
4. **你的底线**（如"可以退但想挽留""坚决不退""可以给一个月免费延期"）

## 工作流程

### 第一步：局势分析

快速判断这封邮件的性质：

| 维度 | 判断 |
|------|------|
| 紧急度 | 🔴高 / 🟡中 / 🟢低 |
| 用户情绪 | 愤怒 / 失望 / 困惑 / 焦虑 / 友好 |
| 对业务影响 | 流失风险 / 差评风险 / 口碑机会 / 升级销售机会 |
| 最佳回应策略 | [一句话策略] |

### 第二步：语气选择

根据用户情绪和场景，选择语气档位：

| 语气 | 适用场景 | 特征 |
|------|----------|------|
| **Empathetic** | 投诉、退款、Bug 影响用户 | 先道歉再解决，多用 "I understand""I'm sorry" |
| **Professional** | 功能请求、使用咨询、商务合作 | 简洁清晰，敬语适度，给确定性 |
| **Friendly** | 好评感谢、老用户续费、社区互动 | 轻松但有分寸，可加一个 emoji |
| **Firm-but-fair** | 无理退款、滥用试用、违反 ToS | 礼貌但不退让，引政策条款 |

### 第三步：邮件结构

按标准客服邮件结构生成：

```
[称呼]

[共情句 — 1 句，承认用户感受]

[解决方案 — 2-4 句，清晰告知你会怎么做/已经做了什么]

[额外价值 — 可选，补偿/优惠/替代方案]

[行动指引 — 用户需要做什么，如果有的话]

[收尾 — 1 句，表达感谢或持续支持的态度]

[签名]
```

### 第四步：关键措辞建议

对回复中的关键节点，给出措辞说明：

- **道歉**：什么时候用 "I apologize" vs "I'm sorry" vs "We regret"
- **拒绝**：怎么在说 "no" 的同时让用户觉得被尊重
- **延迟**：怎么告知用户问题需要时间解决但不让用户觉得被敷衍
- **挽留**：用户说要取消时，怎么给台阶让用户留下来
- **升级**：什么时候把用户引流到更高价的方案

### 第五步：输出最终回复 + 附注

- **可直接发送的邮件全文**（纯文本，方便复制）
- **备选开头**（如果第一版语气不够合适，给一个不同风格的版本）
- **文化提示**：这封回复中需要注意的跨文化沟通细节（如欧美用户 vs 东南亚用户的不同期待）
- **后续跟进**：建议的跟进时间点和措辞

## 常见场景速查

本技能应能覆盖以下高频场景，每种都有预设的应对框架：

| 场景 | 核心策略 |
|------|----------|
| 用户要求退款 | 先共情 → 问原因 → 给替代方案（延期/降级/credits）→ 最后退 |
| Bug 导致用户数据丢失 | 立即道歉 → 说明影响范围 → 告知修复时间线 → 给补偿 |
| 功能请求 | 感谢 → 告知可行性（短期内/roadmap 中/暂不考虑）→ 问使用场景细节 |
| 负面评价威胁 | 私下沟通 → 解决核心问题 → 请用户更新评价（不道德绑架） |
| 续费提醒 | 感谢使用 → 强调这段时间的价值 → 清晰告知续费时间和方式 |
| 试用期即将结束 | 回顾用户使用数据（如有）→ 说明付费后的额外价值 → 给一个小优惠 |
| 用户表扬 | 真诚感谢 → 邀请传播（"if you know anyone who might benefit..."）→ 无强推销 |

```

[可直接发送的全文]

## 备选开头

[不同语气的另一个版本]

## 措辞说明

- 道歉：[解释为什么用了这个词]
- ...

## 文化提示

[跨文化注意事项]

## 后续跟进

- [时间点]：[动作]
```

## 核心原则

- **12 小时内回复**：即使没解决也要告知"收到了，在处理"
- **先解决情绪，再解决问题**：用户愤怒时先共情，不要一上来就解释
- **给确定性**：即使不确定，也要给一个"我什么时候能告诉你更多"
- **挽留但不纠缠**：用户坚决要退就痛快退，留一个好印象比一个月的订阅费重要
- **不编造功能**：不能承诺产品没有的功能
- **语言从简**：非母语客服不要装母语者，清晰的简单英语 > 生硬的高级词汇
